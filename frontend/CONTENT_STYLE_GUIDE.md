# Content Style Guide

This guide standardizes page copy and section structure across the website.

## Core Section Taxonomy

Use these labels consistently for top-level page sections:

- `Methodology`
- `Outcomes`
- `Assurance`
- `Coverage`
- `Next Step`

Avoid introducing alternate labels for the same intent (for example, "Approach", "Framework", "How We Work") unless truly different.

## Heading Pattern

Use enterprise-style sentence patterns:

- `Methodology for ...`
- `Outcomes for ...` or `Outcomes Across ...`
- `Assurance for ...` or `Assurance Through ...`
- `Coverage Across ...`
- `Next Step for ...`

Examples:

- `Methodology for Enterprise AI Delivery`
- `Assurance for End-to-End Solution Reliability`
- `Next Step for Your Industry AI Rollout`

## Supporting Description Pattern

Keep one-sentence supporting copy under section headings.

Preferred phrasing:

- `This methodology ...`
- `This assurance model ...`
- `This next step ...`

Tone:

- Direct, practical, enterprise-focused
- Outcome-oriented, not hype-oriented
- Clear and specific over abstract marketing language

## Recommended Page Flow (Top-Level Pages)

Use this order whenever possible:

1. Hero
2. Core content block(s) (cards/listings)
3. Methodology
4. Outcomes and/or Coverage
5. Assurance
6. Next Step (dark CTA band)
7. Contact section

## Visual/Content Consistency Rules

- Use `Roboto` and `Roboto Mono` for UI typography (see `index.css` and `ubuntu-home.css`).
- Use concise section intros (1 sentence, max 2).
- Keep card titles short and functional.
- Keep card body text outcome-driven and easy to scan.
- Use `STEP 01/02/03...` chips for process sections.

## Navigation Naming

For Company dropdown pages, use:

- `About Us`
- `Leadership`
- `Engagement`
- `Careers`
- `Partners`
- `Security`
- `News & Media`
- `Privacy Policy`
- `Terms and Conditions`

## Do / Don't

Do:

- Reuse existing section patterns before creating new copy styles.
- Keep terminology stable across pages.
- Prefer measurable language (speed, quality, reliability, governance).

Don't:

- Mix multiple naming styles for equivalent sections.
- Use oversized decorative copy that breaks visual rhythm.
- Add generic filler text without business context.
